# 用户系统与数据隔离

Web 认证强制开启后，系统区分两类账号体系：

- 管理后台账号：存储在 `admin_users`，用于访问 `dsa-admin` 和维护平台级配置。
- C 端普通用户：存储在 `users`，在 `dsa-web` 使用用户名注册，注册成功后自动登录。

`admin` 用户名保留给管理后台账号，C 端开放注册不能创建 `admin` 账号。历史 `users.admin` 只用于旧数据归属兼容，不再作为 C 端配置管理入口。

普通用户注册要求：

- 用户名长度至少 3 位。
- 仅支持小写字母、数字、下划线和连字符。
- 密码长度至少 6 位。

## 隔离范围

以下数据按当前登录用户隔离：

- 分析历史和异步分析任务
- 新闻情报和基本面快照
- 问股会话
- LLM 用量统计
- 持仓账户、交易、资金流水、公司行为、持仓快照和风险报表
- C 端个人配置

行情缓存、股票日线、指数和汇率缓存等公共市场数据仍共享，避免重复拉取。

## 旧数据迁移

服务启动时会自动创建 `admin` 用户，并把历史无归属数据归属到 `admin`。已有管理员密码会继续复用；如果历史环境没有保存管理员密码，首次访问时需要设置初始管理员密码。

## 配置说明

平台级配置由 `dsa-admin` 的“系统设置”维护，写入全局 `.env`，包括收费/积分规则、平台默认模型与密钥、平台数据源密钥、系统运行参数、平台 Agent/回测默认参数等。

C 端设置页只维护当前用户自己的配置，写入 `user_configs`。Webhook/通知、自选股、分析偏好、Agent 启用状态、Agent 架构、编排深度、执行预算、深研预算、记忆、策略权重和多 Agent 模型分配等运行偏好属于 C 端个人配置，具体可见和可编辑范围由管理后台角色权限控制。多 Agent 模型分配只能从平台已配置的可用模型中下拉选择；未授权的用户会继续使用平台默认配置，已授权但未设置个人值的字段按空值处理。平台默认 Agent 模型分配由 `dsa-admin` 的“系统设置”维护，同样通过下拉选择 technical/intel/risk/decision 的默认模型；平台模型渠道、API Key、Agent 策略目录和后台事件监控规则仍由 `dsa-admin` 维护。

多进程或多 worker 部署时，会话密钥和账号数据仍需通过共享数据目录或滚动重启保持一致。

## 配置 fallback 规则

普通用户获得某个配置项权限后，该配置项只读取自己的 `user_configs` 值；个人值为空或未保存时按空值处理，不再继承平台配置。普通用户没有权限的配置项才继续继承 `dsa-admin` 维护的平台配置，且会忽略历史残留的个人值。
